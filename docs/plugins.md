# Plugin Integrations

- **Claude Code plugin:** Install from marketplace with `/plugin marketplace add https://github.com/caozhiyuan/copilot-api.git` then `/plugin install claude-plugin@copilot-api-marketplace`. Injects `__SUBAGENT_MARKER__` on subagent starts. `maximal app claude-code --enable` writes both `env.ANTHROPIC_BASE_URL` and an `apiKeyHelper` command into `~/.claude/settings.json`; the helper prints the Claude Code API-client key from Settings → API clients, falling back to the default endpoint key. (The same key is available on demand via `maximal api claude-code`.) Details:
  - **Absolute path, not a bare `maximal`.** The `apiKeyHelper` value is `"<abs-path-to-maximal>" api claude-code` (the running binary's `process.execPath`, double-quoted). Claude Code runs the helper via `/bin/sh -c` (or `cmd.exe`), and a GUI-launched Claude Code inherits a minimal PATH that lacks `~/.local/bin` — a bare `maximal` would fail with `command not found`. The absolute path works on macOS, Linux, and Windows (where there is no `~/.local/bin` symlink at all). A stale path (after an app update/move) — or a config still carrying the legacy `--apiKeyHelper claude-code` form — is recognized as ours by signature and self-healed forward to the current `api claude-code` command on the next boot/apply.
  - **Reverts to the prior state on disable.** Toggling Claude Code off restores `env.ANTHROPIC_BASE_URL` and `apiKeyHelper` to exactly what was there before maximal first wrote them (or removes them if they were absent), via a `_maximalPrior` snapshot maximal stores in the file. A pre-existing *foreign* `apiKeyHelper` / base URL is never overwritten in the first place — enable refuses with a conflict surfaced in Settings → Apps.
- **Opencode plugin:** Copy `.opencode/plugins/subagent-marker.js` to `~/.config/opencode/plugins/`.
