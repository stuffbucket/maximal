copilot-api

A local proxy that exposes GitHub Copilot as OpenAI- and Anthropic-compatible HTTP endpoints. Run it once, point Claude Desktop and other tools at http://localhost:4141.

Internal preview — binaries are unsigned for v1. The first-launch steps below are load-bearing; please follow them.

First launch

macOS: Gatekeeper will say "copilot-api can't be opened because it is from an unidentified developer." Right-click copilot-api.app in /Applications → Open → confirm Open in the dialog. After the first launch, double-click works normally.
Windows: SmartScreen will say "Windows protected your PC." Click More info → Run anyway. The installer registers an at-logon scheduled task; the proxy starts on next sign-in.
Homebrew: Unaffected — brew bypasses Gatekeeper.

Verify it's running

From a new terminal:

copilot-api debug

Should print the resolved port, auth state, and Claude Desktop config status. If the wizard didn't run during install, run copilot-api setup once interactively to finish the GitHub device-code flow.

Uninstall

copilot-api uninstall

Add --purge to also remove the on-disk database and config; --revert-claude to strip the keys this installer added to claude_desktop_config.json.
